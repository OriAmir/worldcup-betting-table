# 🏆 טבלת הניחושים – מונדיאל 2026

טבלת ניחושים משולבת ל"חברים", המשלבת שתי פלטפורמות:

- **365 (דומינוס)** — צילום מצב **קבוע** של הנקודות **עד משחק צרפת–סנגל** (מתוך הצילום מסך).
- **5 חבר'ה (ערוץ הספורט)** — נקודות **חיות**, מצרפת–סנגל והלאה. מתעדכן אוטומטית.

## מודל הניקוד

```
סה"כ = 365 (עד צרפת–סנגל)        ← קבוע, מתוך הצילום מסך
     + 5 חבר'ה (מצרפת–סנגל והלאה) ← חי, מתעדכן כל 30 שניות
     + התאמה ידנית               ← למשל ניר +5.5 (שכח את ניחוש צרפת)
     + בונוס אלופה               ← יחס הקבוצה ב-5 חבר'ה, מתווסף כשתיוודע האלופה
```

- **אלופה / מלך שערים:** אם לשחקן יש בחירה ב־5 חבר'ה — משתמשים בה. אם אין —
  משלימים מ־365 ומסמנים ב־`*` (שורה מודגשת = דורשת השלמה ידנית בסוף הטורניר).
- **בונוס אלופה** = יחס הקבוצה ב־5 חבר'ה (צרפת 30, ברזיל 50, נורווגיה 120 …),
  מתווסף לסה"כ רק כשממלאים את האלופה הרשמית ב־`lib/config.js`.
- **מלך שערים:** 5 חבר'ה כבר נותן 2 נק' לכל גול, חי — כבר כלול בעמודת 5 חבר'ה.

## עריכת נתונים — `lib/config.js`

| שדה | תיאור |
| --- | --- |
| `DOMINOS_SNAPSHOT` | נקודות 365 + ניחושי אלופה/מלך שערים, עד צרפת–סנגל (קבוע) |
| `TEAM_RATIOS` | יחס כל קבוצה ב־5 חבר'ה (בונוס אלופה) |
| `MANUAL_ADJUSTMENTS` | תוספות ידניות (ניר +5.5) |
| `BONUS` | האלופה ומלך השערים הרשמיים בסוף הטורניר |
| `HIDE_PLAYERS` | שחקנים להסתרה (חשבונות דמה) |

## איך מריצים מקומית (לפני push)

```bash
# 1. התקנת תלויות (פעם אחת)
npm install

# 2. בדיקת לוגיקת הניקוד
npm test

# 3א. הרצה מלאה כולל ה-API החי (מומלץ) – דורש Vercel CLI:
npx vercel dev
#     פותח http://localhost:3000 עם /api/standings עובד מול 5 חבר'ה החי.

# 3ב. לחלופין, רק ה-UI (ללא API) :
npm run dev
#     פותח http://localhost:5173 – שימו לב: /api/standings לא יעבוד בלי vercel dev.

# 4. בדיקת הטבלה הממוזגת מול הנתונים החיים, בלי דפדפן:
node -e "import('./lib/sources.js').then(async s=>{const {mergeStandings}=await import('./lib/merge.js');const d=await s.fetchSport5();console.table(mergeStandings(d).rows.map(r=>({rank:r.rank,name:r.name,'365':r.dominosPoints,sport5:r.sport5Points,adj:r.adjustPoints,total:r.total})));})"
```

> `npm run dev` (Vite לבדו) **לא** מריץ את `/api/standings`. כדי לבדוק הכל
> מקומית כמו בפרודקשן השתמשו ב־`npx vercel dev`.

## פריסה ל־Vercel

הריפו כבר מחובר. לפריסה ידנית:

```bash
npx vercel --prod
```

לפריסה אוטומטית בכל commit: ב־Vercel → Project → Settings → Git → חברו את
`OriAmir/worldcup-betting-table`.

## מבנה

```
api/standings.js   Vercel serverless – מושך 5 חבר'ה (ללא CORS), ממזג עם צילום 365
lib/sources.js     fetch של 5 חבר'ה (חי)
lib/merge.js       לוגיקת מיזוג טהורה (נבדקת ביוניט-טסטים)
lib/config.js      כל הנתונים הניתנים לעריכה
src/               אפליקציית React (Vite)
test/              יוניט-טסטים
```
