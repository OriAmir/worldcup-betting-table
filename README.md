# 🏆 טבלת הניחושים – מונדיאל 2026

טבלת ניחושים משולבת ל"חברים", המסונכרנת **בזמן אמת** משתי פלטפורמות:

- **מקור ראשי:** 5 חבר'ה (ערוץ הספורט) — `hevre.sport5.co.il`
- **מקור השלמה:** דומינוס 365 (365scores) — עבור ניחוש **אלופה / מלך שערים** של שחקנים שלא בחרו ב־5 חבר'ה.

## איך זה עובד

- הטבלה הראשית נלקחת מ־**5 חבר'ה** ומתעדכנת אוטומטית כל 30 שניות (ללא מגע יד).
- שחקן שאין לו בחירת **אלופה** או **מלך שערים** ב־5 חבר'ה — הבחירה מושלמת מ־**דומינוס** ומסומנת ב־`*`.
- שורות שדורשות השלמה ידנית בסוף הטורניר מודגשות וצבועות.
- התאמות ידניות (כמו **ניר +5.5** על ניחוש צרפת שנשכח) מוגדרות ב־`lib/config.js`.
- טבלת בונוס נפרדת מציגה את ניחושי האלופה/מלך השערים של כל שחקן.

## הגדרות (`lib/config.js`)

| שדה | תיאור |
| --- | --- |
| `NAME_MAP` | מיפוי שמות בין 5 חבר'ה ↔ דומינוס |
| `MANUAL_ADJUSTMENTS` | תוספות/קיזוזים ידניים (למשל ניר +5.5) |
| `BONUS` | תשובות רשמיות לאלופה/מלך שערים + ניקוד בסוף הטורניר |
| `HIDE_PLAYERS` | שחקנים להסתרה (חשבונות דמה) |

בסיום הטורניר מלאו את `BONUS.correctWinner` ו־`BONUS.correctTopScorer` והנקודות יתווספו אוטומטית.

## ארכיטקטורה

```
api/standings.js   → Vercel serverless: מושך את שתי הפלטפורמות (ללא CORS), ממזג, ומחזיר JSON
lib/sources.js     → fetch של 365scores + sport5
lib/merge.js       → לוגיקת המיזוג הטהורה (נבדקת ביוניט-טסטים)
lib/config.js      → כל ההגדרות הניתנות לעריכה
src/               → אפליקציית React (Vite)
test/              → יוניט-טסטים ללוגיקת המיזוג
```

## הרצה מקומית

```bash
npm install
npm test          # בדיקת לוגיקת המיזוג
npm run build     # build לפרודקשן

# להרצה מלאה עם ה־API (דורש Vercel CLI):
npx vercel dev
```

> הערה: `npm run dev` של Vite לבדו לא מריץ את פונקציית ה־API. השתמשו ב־`vercel dev` כדי לבדוק את `/api/standings` מקומית.

## פריסה ל־Vercel

1. דחפו את הריפו ל־GitHub.
2. ב־[vercel.com](https://vercel.com) → **Add New → Project** → ייבאו את הריפו.
3. Vercel מזהה Vite אוטומטית. לחצו **Deploy**. זהו.

האתר יתעדכן בזמן אמת מול שתי הפלטפורמות ללא צורך בהתערבות ידנית.
